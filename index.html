Great! With your PayPal Business account, you can automatically add players to your tournament when they complete payment.

Step 1: PayPal Developer Dashboard Setup

  1. Go to developer.paypal.com
  2. Log in with your PayPal Business account
  3. Create a new app or edit existing app
  4. Note your Client ID and Client Secret

Step 2: Webhook Configuration

In your PayPal app dashboard:

  1. Go to "Webhooks" section
  2. Click "Add Webhook"
  3. Enter your webhook URL:
https://your-username.github.io/tournament-manager/webhook

Subscribe to these events:

  • PAYMENT.SALE.COMPLETED
  • PAYMENT.SALE.PENDING

Step 3: Update Your Registration Form

Step 4: Webhook Handler Options

Since GitHub Pages only hosts static files, you have these options for webhook handling:

Option A: Vercel Serverless Functions (Recommended)

  1. Create a Vercel account (free)
  2. Deploy webhook handler as serverless function
  3. Use Vercel's database integration

Option B: Netlify Functions

  1. Create Netlify account (free)
  2. Deploy webhook as Netlify function
  3. Use Netlify's database or external service

Option C: Manual Processing (Fallback)

Check PayPal transaction history and manually add players using the tournament manager interface.

Step 5: Testing

  1. Use PayPal Sandbox for testing
  2. Create test payments
  3. Verify webhook receives data
  4. Check players are added to tournament

Test webhook with sample data using the "Test PayPal" button in the tournament manager.

Important: Store your PayPal credentials securely and never expose them in client-side code. Use environment variables in your webhook handler.

Step 6: Go Live

  1. Switch from Sandbox to Live in PayPal Developer Dashboard
  2. Update webhook URLs to production
  3. Test with real (small) payments
  4. Monitor webhook logs for any issues
